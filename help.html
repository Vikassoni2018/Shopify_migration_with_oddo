Find answers for uploads, Shopify access, jobs, and Render deployment.

This page is tailored to the real behavior of your project, including local `.env` runtime config, Shopify Admin API connections, CSV preparation, job tracking, and admin setup.

Need quick direction?
  • Use `/odoo-migration` for order-focused Matrixify preparation.
  • Use `/woocommerce-migration` for product CSV conversion and direct import.
  • Use `/admin` to verify environment and deployment readiness.
CSV

Upload guidance

Make sure exports are true CSV files and not spreadsheet links or partial tabular copies.

API

Shopify credentials

Use the `your-store.myshopify.com` domain and a valid Admin API token with the scopes required by the route you are using.

ENV

Local vs Render

Local uses `.env`; Render uses environment variables from the dashboard or blueprint.

Frequently asked questions

Answers grounded in this project’s current behavior

Open `/odoo-migration`, connect your Shopify store, upload the Odoo CSV, preview the transformed data, then either download the generated files or start the import job directly from the interface.
The dedicated WooCommerce page accepts a product export CSV, converts the fields to Shopify format, and can call the server endpoint that creates or updates Shopify products. Public image URLs in the `Images` column can also be attached during import.
Push the repository with `render.yaml`, then set environment values like `ADMIN_EMAIL`, `ADMIN_PASSWORD`, and any Stripe keys you want to use. The app already reads `process.env.PORT` and serves runtime config to the frontend.
Create or update `.env`, set a free `PORT` such as `3460`, point `LOCAL_API_BASE_URL`, `DEFAULT_API_BASE_URL`, and `PUBLIC_BASE_URL` to that local address, then run `npm start`.
Admin login is server-backed. Credentials are read from environment variables, and the browser receives a signed session cookie instead of storing the admin password locally.
